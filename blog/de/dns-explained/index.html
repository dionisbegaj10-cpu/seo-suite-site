DNS erklärt: Das System hinter jeder Website-Adresse

Jedes Mal, wenn Sie eine Website-Adresse in einen Browser eingeben, übersetzt etwas diesen menschenlesbaren Namen in die numerische Adresse, die Computer tatsächlich zur gegenseitigen Kommunikation nutzen. Dieses Übersetzungssystem ist DNS, das Domain Name System, vergleichbar mit einem Telefonbuch, das einen Namen in eine Nummer übersetzt.

A-Records verweisen eine Domain auf eine spezifische IPv4-Adresse, der grundlegendste Recordtyp. CNAME-Records verweisen eine Domain oder Subdomain auf eine andere Domain statt direkt auf eine IP-Adresse. MX-Records bestimmen, wohin E-Mails für eine Domain zugestellt werden, getrennt vom Website-Hosting. TXT-Records enthalten beliebige Textdaten, häufig für Domainverifizierung und E-Mail-Authentifizierung wie SPF und DKIM genutzt.

Wird ein DNS-Record aktualisiert, etwa um eine Domain auf einen neuen Host zu verweisen, tritt diese Änderung nicht sofort überall in Kraft, DNS-Informationen sind an verschiedenen Stellen im Internet zwischengespeichert, eine vollständige globale Verbreitung kann bis zu 48 Stunden dauern.

DNS-Records werden meist entweder über den Domain-Registrar oder einen separaten DNS-Anbieter verwaltet, zu wissen, wo Ihr DNS tatsächlich verwaltet wird, ist essenziell, bevor Sie Änderungen vornehmen, sonst hat die Bearbeitung an der falschen Stelle keinerlei Effekt auf die live geschaltete Seite.

Häufig gestellte Fragen

Warum dauert es, bis DNS-Änderungen live sind?

DNS-Einträge werden an vielen Stellen im Internet zwischengespeichert, und diese Caches müssen erst ablaufen, bevor alle die Änderung sehen — das ist normal und kein Zeichen für ein Problem.

Was passiert, wenn ich DNS-Einstellungen falsch setze?

Meist wird die Website oder E-Mail bis zur Korrektur unerreichbar — DNS-Fehler verursachen selten dauerhaften Schaden, können aber echte Ausfallzeit bedeuten, daher lohnt sich eine Kontrolle vor dem Speichern.
